# 용어 감수 체크리스트 (Terminology Audit)

VC 도메인 제품의 spec·UI 카피·i18n·코드 식별자에서 용어가 정확하게 쓰였는지 감수할 때 사용한다. 정의는 `07-glossary-and-kr-regulatory.md` 기준.

## 감수 절차

1. **대상 정하기** — 기획 문서(spec) → UI 카피(i18n/카피 리소스) → 코드 식별자(상태 enum·분류 코드) 순으로 우선순위.
2. **아래 혼동 쌍 표를 grep으로 훑는다** — 각 행의 "혼동/오용" 쪽 패턴을 검색해 히트를 하나씩 눈으로 검증한다 (기계적 치환 금지 — 문맥상 맞는 사용도 있다).
3. **판정 기준**: (a) 용어 자체가 틀림(오용) / (b) 맞지만 제품의 확정 용어와 불일치(표기 통일 필요) / (c) 문맥상 맞음(false positive).
4. 수정은 **문서와 UI 카피를 같은 변경에서** — 문서만 고치고 UI를 남기면 다시 어긋난다.
5. 제품별 확정 용어(페르소나 명칭, deprecated 용어 목록)가 있으면 그 목록을 이 절차의 0순위 필터로 추가한다.

## 자주 혼동되는 용어 쌍 (Confusion Pairs)

| ✅ 올바른 용어 | ❌ 혼동/오용 | 구분 기준 |
|---|---|---|
| 운용사 (GP) | 운영사 | **한 글자 차이.** 펀드를 굴리는 GP는 "운용사". "운영사"는 서비스·시설을 운영하는 회사 — 다른 말 |
| 신주인수계약 (투자계약) | 주식매매계약(SPA) | 신규 투자 = **신주** 발행·인수. SPA는 **구주**(세컨더리) 매매. 신규 라운드 문서를 SPA로 부르면 부정확 |
| 텀시트 (비구속) | "계약" 취급 | 텀시트는 non-binding. 구속력은 본계약(신주인수계약/SHA)에서. "텀시트 체결로 투자 확정" 류 카피는 오류 |
| Pre-money | Post-money | 지분율 = 투자금 ÷ **post-money**. 밸류에이션 입력 필드는 어느 쪽인지 반드시 명시 |
| 투자심의위원회 (투심위) | 투자심사위원회 | 표준 관행 용어는 "심의". IC 번역도 투자심의위원회로 통일 |
| 성과보수 (carry) | 성과급 | 펀드 GP의 carried interest는 "성과보수" |
| 운용보수 | 관리보수/수수료 | management fee의 표기를 제품 내 하나로 통일 (국내 규약은 "관리보수"도 씀) |
| 우선수익률 (hurdle) | 목표수익률 | hurdle(carry 문턱)과 target IRR(펀드 목표)은 다른 개념. 펀드 생성 폼에 둘 다 있으면 라벨 구분 |
| 수익률 | 수익율 | 맞춤법: 받침 뒤는 "률"(수익률·성장률), 모음/ㄴ 뒤는 "율"(이자율·환율) |
| 청산우선권 | 우선매수권 | liquidation preference(청산 시 우선 회수) vs ROFR(지분 매각 시 우선 매수) — 전혀 다른 권리 |
| 신주인수권(주주 preemptive) | 신주인수권부사채(BW)의 워런트 | 같은 "신주인수권"이라도 주주권 vs 사채 워런트 |
| 동반매도**요구**권 (drag) | 동반매도**참여**권 (tag) | drag=강제로 끌고 감(다수→소수), tag=따라감(소수의 권리) |
| 상환권 (redemption) | 환매/풋옵션 | RCPS의 상환권은 배당가능이익 한도 내 — 무조건 환매처럼 쓰면 오류 |
| 캐피탈 콜 (수시납) | 일시납 | 결성 시 전액 납입(일시납)과 수시 납입 요청은 다른 구조. 펀드 모델링 시 어느 쪽인지 명시 |
| 결성총액 (약정) | 납입액 | committed ≠ contributed. 소진율·dry powder 계산의 분모 주의 |
| 좌 / 출자좌수 | 구좌 | 법령·규약 용어는 "좌"/"출자좌수". "구좌"는 구어 — 제품 카피는 하나로 통일하고 계약·규약 문맥에선 "좌" |
| 좌수 (수량) | 금액 | 조합형 펀드의 약정·의결권·배분은 좌수 기준. 좌 단위 펀드에서 금액만 저장/표시하면 배분 계산과 어긋남 |
| DPI | TVPI/MOIC | "회수 배수"라고만 쓰면 모호 — 실현(DPI)인지 총가치(TVPI)인지 |
| 완전희석 기준 | 발행주식 기준 | 지분율 표시는 fully diluted 여부를 명시해야 캡테이블 수치가 안 어긋남 |
| 실사 (DD) | 심사 | 국내 "심사"는 스크리닝+실사를 아우르는 넓은 말. 문서 검증 행위 자체는 "실사" |
| 포트폴리오사 | 피투자사/투자기업 | 제품 내 한 가지 표기로 통일 |
| 조건부지분인수계약 | SAFE(영문 그대로) | 국내 법정 명칭은 조건부지분인수계약 — UI에선 "SAFE(조건부지분인수계약)" 병기 권장 |
| 주식매수선택권 | 스톡옵션 | 법정 용어는 주식매수선택권. 캐주얼 카피엔 스톡옵션 허용하되 계약 문맥에선 법정 용어 |
| IRR (Net) | IRR (Gross) | LP에게 보여주는 성과는 보수 차감 후(Net)가 기본 — 어느 쪽인지 라벨 명시 |
| 데이터룸 계층 용어 | 상·하위 개념 혼용 | 제품이 정의한 계층(예: 파일 집합 vs 공유 단위)이 있으면 상하위를 섞어 쓰지 않는다 |

## 감수용 grep 스니펫

경로는 대상 프로젝트에 맞게 조정한다.

```bash
# 률/율 맞춤법
grep -rn "수익율\|성장율\|증가율률" <docs> <ui-src>
# SPA 오용 후보 (신규투자 문맥인지 확인)
grep -rn "주식매매계약\|SPA" <docs> <ui-src>
# 운용사/운영사 구분 (GP 문맥에 "운영사"가 쓰였는지)
grep -rn "운영사" <docs> <ui-src>
# pre/post money 라벨 명시 여부
grep -rn "기업가치\|밸류에이션\|valuation" <ui-src>
# 텀시트를 구속 계약처럼 쓰는 카피
grep -rn "텀시트.*체결\|텀시트.*확정" <docs> <ui-src>
```
